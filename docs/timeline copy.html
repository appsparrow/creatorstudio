Creator Studio

How one person built the AI influencer content platform in 2 weeks

By Siva Tayi · March 2026 · Built with Claude Code

The Problem

Three pain points every AI content creator knows too well

🪞

The Consistency Crisis

AI-generated faces change between every post. Your "influencer" looks like a different person every time.

🧩

The 6-Tool Nightmare

ChatGPT + Midjourney + Kling + Canva + Later + Instagram. Six tools, six tabs, zero flow state.

📢

Content Without Purpose

No audience strategy, no pain-point targeting. Just pretty pictures shouting into the void.

The Build Timeline

13 days, from idea to production

Days 1-3

Phase 1: The Prototype

Started with raw Express + SQLite + React. No auth, no cloud, no design system. Just a local app that could define a persona, generate images with NanoBanana and Gemini, create videos with Kling AI, and post to Instagram via Blotato.

"The tool shouldn't just generate content — it should understand WHO the content is for."
Day 4

The Pivot

Threw away 3,144 lines of code. Not because it was broken — it worked great. But it was a monolith. One file. No auth. Local only. If this was going to be a real product, it needed real architecture.

3,144 lines deleted
Days 4-5

Phase 2: New Architecture

Supabase PostgreSQL with Row Level Security. Cloudflare Workers (Hono) as the API layer. Cloudflare R2 for media storage. Multi-tenant from day one.

Zero egress fees
Day 5

The Canvas Moment

Built a beautiful multi-page SaaS app with routing and navigation. Used it for 10 minutes and hated it. Too many clicks. Too much context switching. Rebuilt as a single-screen workspace.

"This should feel like a canvas, not an admin panel. Everything in one view. Click to edit, not click to navigate."
Days 6-7

Phase 3: AI-First

The biggest design decision: AI fills everything, human tweaks. Describe what happens and the system generates theme, scene, caption, hashtags, CTA, hook, music suggestion, location, story arc, and caption tone.

80% less manual work
Days 7-10

Phase 4: Audience Strategy

Every post now connects Persona (WHO), Target Audience (FOR WHOM), Content Focus (ABOUT WHAT), and Products (WITH WHAT). Up to 6 audience segments with pain points, aspirations, and resonance notes.

"Content without audience awareness is noise."
Day 8

Friends & Storylines

Recurring characters that appear in your persona's storyline. Each friend has a photo, traits, relationship, and an active/inactive toggle. Diversity rules enforced automatically.

"Your AI influencer's best friend should look the same in post 1 and post 100."
Day 9

Content Types

Photo (scene + text overlay), Carousel (4+ slides with sequential storyline), Video (hook, thumbnail, camera angle, audio). One AI prompt generates content for all three types.

1 prompt → 3 formats
Day 11

Deployed

Frontend on Cloudflare Pages, API on Cloudflare Workers, database on Supabase PostgreSQL, media on R2, auth on Supabase Auth. All media files migrated from local filesystem.

64 files migrated to R2
Day 12

Thumbnail Generation

Upload reference thumbnails — the poses, expressions, and framing you want — and the AI generates click-worthy thumbnails in that style. Because the thumbnail determines whether anyone watches your video.

Day 13

Google Drive Integration

Per-persona Drive folders. Sync files, select images and videos for posts, mix Drive media with AI generation. For carousels: pick 2 from Drive, generate the other 2.
